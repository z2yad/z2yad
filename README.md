## 🚀 Featured Projects

### 🛒 Rendering — Angular E-Commerce Platform

<a href="https://e-commerce-iota-sand.vercel.app/"><img src="https://img.shields.io/badge/🔗_Live_Demo-e--commerce--iota--sand.vercel.app-000000?style=flat-square&logo=vercel"/></a>
&nbsp;
<a href="https://github.com/z2yad/E-commerce"><img src="https://img.shields.io/badge/Source_Code-GitHub-181717?style=flat-square&logo=github"/></a>
&nbsp;
<img src="https://img.shields.io/badge/Status-In_Progress-f59e0b?style=flat-square"/>

> A production-level e-commerce platform demonstrating full-stack engineering maturity — from architecture decisions to UI micro-interactions.

<details>
<summary><b>⚡ Technical Deep-Dive (click to expand)</b></summary>
<br/>

**🔴 Problem**
Modern e-commerce needs fast initial loads for SEO, reactive state for cart/checkout UX, and a maintainable codebase that can scale from prototype to production.

**🟡 Solution**
- **SSR via `@angular/ssr`** — server-renders critical routes for SEO-optimized page loads
- **Per-route rendering strategy** — SSR for product listing/detail (SEO), CSR for cart/checkout (interactivity)
- **Reactive cart service** — powered by Angular Signals + RxJS for real-time state across components
- **spartan/ui + Tailwind CSS 4** — OKLCH color spaces, glassmorphism, fully custom design tokens
- **Route guards on checkout** — prevents invalid state navigation, ensuring data integrity
- **PrimeNG admin panel** — feature-rich dashboard with minimal custom code

**🟢 Impact**
Deployed to Vercel with measurable SSR performance. Architecture is migration-ready for Spring Boot backend, replacing the current JSON Server mock API.

</details>

<br/>

| Layer | Technology | Purpose |
|:---|:---:|:---|
| Framework | Angular 21 | Component architecture, routing, DI |
| Rendering | `@angular/ssr` | SEO + performance |
| Styling | Tailwind CSS 4.2 + spartan/ui | Design system tokens |
| Notifications | `ngx-sonner` | Toast feedback layer |
| Testing | Vitest 4.1 | Unit test coverage |
| Deployment | Vercel | CI/CD + edge deployment |

### 🧠 QuizMaster — Secure Quiz Engine

<a href="https://github.com/z2yad/QuizMaster"><img src="https://img.shields.io/badge/Source_Code-GitHub-181717?style=flat-square&logo=github"/></a>
&nbsp;
<img src="https://img.shields.io/badge/Status-Live-22c55e?style=flat-square"/>

> A full-featured SPA quiz platform with a security-first authentication system — built entirely in Vanilla JS, demonstrating core JavaScript mastery without framework abstraction.

<details>
<summary><b>⚡ Technical Deep-Dive (click to expand)</b></summary>
<br/>

**🔴 Problem**
Build a dynamic, multi-category quiz app with secure auth and state management — without any framework.

**🟡 Solution**
- **SHA-256 + unique salt hashing** via Web Crypto API — password security without a backend
- **SPA routing** with Navigo 7 — zero-reload navigation with hash routing
- **OpenTDB API integration** — dynamic questions across 6 categories with error handling
- **Light/Dark mode** — CSS custom properties toggle with LocalStorage persistence
- **Responsive breakpoints** — mobile-first grid from 320px to large displays

**🟢 Impact**
Demonstrates framework-agnostic JavaScript skills, cryptographic security awareness, and clean SPA architecture — all valued signals in a senior interview.

</details>

<br/>

| Capability | Detail |
|:---|:---|
| Auth Security | SHA-256 + salt via Web Crypto API |
| Navigation | SPA routing (Navigo 7) — zero reload |
| Data Source | OpenTDB API — 6 categories, dynamic difficulty |
| UI | Light/Dark mode, fully responsive (4 breakpoints) |
